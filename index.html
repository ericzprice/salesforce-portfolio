01

Add a Screen Flow for Debt Intake

Replace manual record creation with a guided Screen Flow that walks an admin through capturing debt type, severity, component, and recommended fix — with validation at each step.

1
Open Flow Builder

Go to Setup → search "Flows" → click New Flow → choose Screen Flow.

2
Screen 1 — Basic Info

Add a Screen element. Drop in these components: a Text input for "Component Name", a Picklist for "Debt Type" (Flow, Validation Rule, Page Layout, Lightning Page, Dashboard, Permission Set, Experience Cloud, Report, Field), a Picklist for "Severity" (Low, Medium, High, Critical), and a Text Area for "Issue Description".

3
Screen 2 — Remediation Details

Add a second Screen. Include a Text Area for "Recommended Fix", a Text input for "Related Object", a Date picker for "Target Resolution Date", and a Checkbox for "Active Issue" (default = checked).

4
Screen 3 — Assignment

Add a Lookup component pointing to the User object for "Assigned Admin". Add a Text Area for "Business Impact" (optional but boosts scoring).

5
Create Record element

After the last screen, add a Create Records element. Set Object = Technical_Debt_Item__c. Map each screen input variable to the matching field. Set Status = "New" as a hardcoded default.

6
Add a Success Screen

Add a final Screen element with a Display Text component: "✅ Technical Debt Item created! Record ID: {!recordId}". This confirms the record saved.

7
Activate & add to the Lightning App

Save and Activate the Flow. Then go to App Builder → open the Technical Debt Command Center page → drag a Flow component onto the page → select your new flow. Save and Activate the page.

💡 Pro tip: Add a "Decision" element after Screen 1 — if Severity = Critical, route to a branch that also sends an email alert to the Assigned Admin. Shows conditional logic to interviewers.

02

Record a Walkthrough Video

A 3–5 minute Loom showing the Command Center → drill into a Critical item → resolve it → watch the dashboard update. Recruiters watch video before reading code.

1
Install Loom (free)

Go to loom.com → sign up free → install the Chrome extension or desktop app. Choose "Screen + Camera" so your face appears in the corner — it builds trust with hiring managers.

2
Plan your 5-minute script (don't wing it)

0:00–0:30 — Intro: "I'm Eric Price, Salesforce Admin. This is a Technical Debt Tracker I built in a Developer Org to show platform governance skills."
0:30–1:30 — Command Center overview: show the two donut charts and the Highest Risk Items panel.
1:30–2:30 — Click into TDI-0004 (Critical). Walk through the fields: Issue Description, Recommended Fix, Risk Score formula breakdown.
2:30–3:30 — Change Status to "Fixed", set Date Resolved to today. Save.
3:30–4:30 — Refresh the Dashboard. Show the Critical Items Open count drop. Show the Fixed Items This Month chart update.
4:30–5:00 — Close: "I'm available for admin, consultant, or platform engineer roles. Link to my GitHub is in the description."

3
Record in one take (it's fine if imperfect)

Close Slack and email. Full-screen your Salesforce org. Hit record. Narrate what you're clicking and why it matters — "I set this to Critical because it affects every resident record" beats silently pointing at fields.

4
Copy your Loom link

Once uploaded, click Share → copy the link. The video is auto-hosted by Loom — no upload needed.

5
Add the link to this portfolio page

In index.html, find the hero CTA buttons and add a third button:

<a href="YOUR_LOOM_URL" target="_blank" class="btn btn-outline"> ▶ Watch Demo </a>
6
Add the link to your resume

Under the Technical Debt Dashboard project on your resume, add a line: Demo video: [your loom URL]. Recruiters open this before they read bullet points.

03

Push Source to GitHub with a README

Deploy metadata via Salesforce CLI and commit it to your GitHub repo. Add a README with screenshots and a "why I built this" so interviewers can explore without a demo call.

1
Authenticate your org with Salesforce CLI

Open VS Code terminal (or any terminal). Run:

sf org login web --alias NexGenLiving

Log in when the browser opens. You should see "Successfully authorized".

2
Create a project folder & pull metadata
sf project generate --name TechnicalDebtTracker cd TechnicalDebtTracker sf project retrieve start \ --metadata CustomObject:Technical_Debt_Item__c \ --metadata Flow \ --metadata LightningPage \ --metadata Report \ --metadata Dashboard \ --target-org NexGenLiving

This pulls your custom object, flows, lightning pages, reports, and dashboards into a local force-app/ folder.

3
Initialize Git and push to GitHub
git init git add . git commit -m "Initial commit: Technical Debt Tracker metadata" git remote add origin https://github.com/ericzprice/NexGenLiving.git git push -u origin main
4
Create a README.md in the repo root

Create a file called README.md with this structure:

# Technical Debt Tracker — Salesforce Portfolio Project ## What it does Tracks and prioritizes configuration risks across a Salesforce org using custom objects, formula-based risk scoring, Screen Flows, reports, dashboards, and Lightning App Builder. ## Why I built it [2–3 sentences in your own words] ## Demo [Paste your Loom link here] ## Screenshots ![Command Center](screenshots/command-center.png) ![Dashboard](screenshots/dashboard.png) ![Item Detail](screenshots/item-detail.png) ## Tech used Lightning App Builder · Flow Builder · Custom Objects · Formula Fields Reports & Dashboards · Permission Sets · Salesforce CLI · GitHub
5
Add a screenshots/ folder to the repo

Create a screenshots/ folder in the project. Copy your 5 PNG files into it. Then:

git add screenshots/ README.md git commit -m "Add README and screenshots" git push
6
Enable GitHub Pages to host this portfolio

Copy index.html and all PNG files into the repo root (or a /docs folder). In GitHub → repo Settings → Pages → set Source to main branch / root (or /docs). Your portfolio will be live at ericzprice.github.io/NexGenLiving in ~2 minutes — a real URL you can put on your resume.

💡 Pro tip: Pin this repo on your GitHub profile (click the pin icon on your profile page). It's the first thing a hiring manager sees when they click your GitHub link from your resume.
